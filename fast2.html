Fast2

Web app to substitute current native software and the accompanying apps.

Purpose

Translation of an old software to a web app. The aim is to able to completely replace the current applications and native software for a web app.

The main challenge is to create a user friendly app from a complex system, without losing any of the features and functions. Also considerations for responsive sizing are needed as both mobile and desktop versions will be used.

Work

I created and managed a design system library, using as a primary the main colour of the company. I created the components and base layouts, including menus and the layout guidelines for the responsive size breakpoints so they could be used throughout the product in a consistent way.



I worked in the development of different features and elements decided by the product owner and different stakeholders. These features are designed in Figma, usually in an interactive prototype to test with the end users. The screens showcase are both in mobile size and desktop size.

In the day to day I was in constant communication with the development team, either to handover new designs or help with any doubts or new issues that might arise. I also have direct contact with the product owner so we can discuss requirements, user feedback, and new features.

Methods used

The main methods I used for this project were:

  • Research of design patterns
  • Analysis of requirements
  • Design showcase
  • Wireframing
  • Prototyping
  • Agile methodology
